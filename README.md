# Запуск проекта

## Сбилдить образ проекта
```console
sudo docker compose build
```

## Создать файл .env в корне проекта
```console
touch .env
```
Заполнить его данными, которые пришлю

## В корне проекта запустить контейнеры
```console
sudo docker compose up
```

## Backend
Работа с приложенем реализована через swagger(для простоты реализации с упором на серверную часть) по адресу [localhost:8000/docs](http://localhost:8000/docs)

## Очистить базу и выключиь контейнеры
```console
sudo docker compose down -v
```
Или нажав ctrl+c

# Основная структура проекта

## ./src/route
Фалы-эндпоинты:
- auth - регистрация/вход/выход
- content - данные, которые можно получить, имя корректную роль
- role - операция с ролями(просмотр/добавление/удаление)

## ./src/db
Директория для работы с БД
### config
- engine.py - движок для создания асинхронных сессий
- url.py - формирование url для подключения к БД
### model
- base.py - базовая модель таблиц
- role.py - модель таблицы "role"
- user_role.py - модель таблицы "user_role"
- user.py - модель таблицы "user"
### service
- role.py - класс для работы с таблицей "role"
- user_role.py - класс для работы с таблицей "user_role"
- user.py - класс для работы с таблицей "user"

## ./src/auth
- password.py - генерация и проверка хеш-пароля
- token.py - класс для работы с JWT

## ./sql
- init.py - базовая структура БД при первом запуске контейнера PostgreSQL
